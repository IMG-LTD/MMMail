# MMMail Support

MMMail Free is a public self-hosted baseline. Support is handled through public issue triage unless a report contains sensitive security details.

- Commercial support boundary: `docs/commercial/support-policy.md`

- Module maturity matrix: `docs/open-source/module-maturity-matrix.md`
- Current release notes: `docs/release/v2.1.2-shipping-clean-release-notes.md`

## Public Issues
- Bugs in the current public baseline: `.github/ISSUE_TEMPLATE/bug-report.md`
- Release-blocking regressions: `.github/ISSUE_TEMPLATE/release-blocking-regression.md`
- Self-hosting install, upgrade, backup, restore, or runbook feedback: `.github/ISSUE_TEMPLATE/self-hosting-feedback.md`
- Non-blocking feature requests: `.github/ISSUE_TEMPLATE/feature-request.md`

## Security Reports

Do not open a public issue for a live security vulnerability.

Follow `SECURITY.md`. Use GitHub private vulnerability reporting whenever possible. If that path is unavailable to you and you do not already have a private maintainer contact path, open only `.github/ISSUE_TEMPLATE/security-contact-request.md` without vulnerability details, proof of concept, logs, credentials, affected endpoints, or reproduction steps.

## Response Targets

Response targets are best effort, not SLA commitments. Current targets are recorded in `MAINTAINERS.md`.
