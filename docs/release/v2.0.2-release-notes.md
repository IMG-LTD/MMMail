# MMMail v2.0.2 Release Notes

Prepared as a focused closure follow-up to the shipped `v2.0.0` baseline.

## Highlights
- close frontend route compatibility evidence for same-shape legacy folder detail handling
- make collaboration and notifications aggregation reads consistently scope-aware
- promote billing readiness into the frozen backend v2 contract regression gates
- align release and support docs with the `v2.0.2` hardening delta

## Validation
- `pnpm --dir frontend-v2 exec node --test tests/foundation-route-contract.test.mjs tests/redirect-contract.test.mjs tests/workspace-aggregation-contract.test.mjs`
- `mvn -f backend/pom.xml -pl mmmail-server -am -Dtest=BillingReadinessIntegrationTest,PlatformCapabilityIntegrationTest,PublicShareCapabilityIntegrationTest,WorkspaceAggregationIntegrationTest,AiMcpCapabilityIntegrationTest,RequestHeaderContractIntegrationTest,ObservabilityIntegrationTest -Dsurefire.failIfNoSpecifiedTests=false test`
- `bash scripts/validate-local.sh`

## Included
- `/folders/:folderId` compatibility is now recorded as a same-shape canonical `/folders/:id` contract instead of a dead redirect claim
- collaboration and notifications now both fetch workspace aggregation under the active org/scope headers
- billing readiness keeps a stable authenticated payload and is covered by the local and CI fast regressions

## Upgrade Notes
- `v2.0.2` stays within the `v2.0.0` shipped boundary and does not add new top-level routes or billing ledger behavior
- existing `v2.0.0` and `v2.0.1` release notes remain authoritative for earlier hardening slices

## Known Limitations
- `v2.0.2` does not expand billing into a ledger, payments, or invoice orchestration release
- `v2.0.2` does not promote Labs modules into new GA top-level products
