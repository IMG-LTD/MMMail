# MMMail v2.0.0 Release Notes

**版本**: `v2.0.0`  
**日期**: `2026-04-21`  
**状态**: `local release material prepared; pending G-release signoff and remote release execution`

## Summary
- `MMMail v2.0.0` closes the frozen M0-M7 upgrade program into a locally validated release candidate.
- This document records landed scope, current ops handoff pointers, the `2.x Roadmap`, and `Known Limitations`.
- It does not claim that `release/2.0.0 → main`, tag push, or GitHub Release publication has already happened.

## 已落地清单 / Landed Scope
- `M0` — release scaffold, progress ledger, contract inventory, and domain / tenant / gateway / flag / Labs / event matrices are committed.
- `M1` — backend v2 module reactor, canonical routes, redirect registry, and org/scope header contract are in place.
- `M2` — platform capability baseline and shared `scope / dialog / soft-auth / async action` contracts are wired for v2.
- `M3` — Mail / Drive / Pass public-share surfaces converge on one shared security and unlock flow.
- `M4` — AI platform and MCP registry contracts are exposed, and core workspace surfaces consume the shared AI / MCP contracts.
- `M5` — workspace aggregation plus Docs / Sheets / Collaboration / Notifications shared-surface integration is landed.
- `M6` — billing readiness contract and approved legacy exits are recorded; redirect baseline is frozen for v2.
- `M7` — Labs disposition evidence is committed, the release blocker checklist is fully checked, and `frontend-v2` validation is wired into both local and CI gates.

## Known Limitations
- Labs modules remain preview or retire-candidate surfaces; `Authenticator`, `SimpleLogin`, `Standard Notes`, `VPN`, `Meet`, `Wallet`, and `Lumo` are not promoted to GA v2 top-level products.
- Billing in `v2.0.0` is a readiness/governance baseline, not a full external payment or detailed ledger release.
- Docs / Sheets stay on the current collaboration baseline; this release does not claim a heavy real-time collaboration engine upgrade.
- This document records local release preparation only and does not claim that remote merge / tag / GitHub Release actions have already been executed.
- The release gate is mechanism-complete locally; it does not claim a real production canary cutover.

## 2.x Roadmap
The following items are explicitly out of scope for `v2.0.0` and remain part of the 2.x line:
- Labs 模块真正产品化实现（Meet / VPN / Lumo / Wallet / Standard Notes parity 仅保留 preview 壳层 + route plumbing）
- `database-per-tenant` 升级（backend §11.1 第 2 项）
- Kafka / 独立 MQ 替换 Redis Streams（backend §11.1 第 5 项）
- Billing ledger 精细模型、外部支付编排（backend §11.1 第 3 项）
- Docs/Sheets 实时协作引擎重型升级（backend §11.1 第 4 项）
- 真实生产环境金丝雀切流（用户确认的 Q4-A 决策）
