# MMMail v2.0.0 Release Notes

Released on `main` as tag `v2.0.0`. The existing GitHub Release `v2.0.0` should match this document.

## Highlights
- backend multi-module scaffold for v2 domains
- frontend-v2 canonical routes, redirects, and shared interaction contracts
- public-share baseline, AI/MCP capability contracts, and aggregation surfaces
- legacy Nuxt surface reduced to approved redirects and preview stubs
- preview modules remain available under `/labs/:moduleKey`

## Validation
- `bash scripts/validate-local.sh`
- `pnpm --dir frontend-v2 test`
- `pnpm --dir frontend-v2 typecheck`
- local and CI release gates include `frontend-v2`

## Upgrade Notes
- legacy compatibility redirects remain in place for the approved routes
- preview modules remain under `/labs/:moduleKey`
- `v2.0.0` shipped on `main` as tag `v2.0.0`; `release/2.0.0` remains the maintenance branch for any post-release alignment work

## Ops Handoff / Runbook Pointers
- `docs/ops/runbook.md`
- `docs/ops/install.md`
- `docs/ops/upgrade.md`
- `docs/ops/backup-restore.md`
- `docs/ops/team-enablement.md`

## Known Limitations
- Labs modules remain preview or retire-candidate surfaces and are not promoted to GA v2 top-level products
- Billing in `v2.0.0` is a readiness/governance baseline, not a full external payment or detailed ledger release
- Docs / Sheets stay on the current collaboration baseline and do not claim a heavy real-time collaboration engine upgrade
- release merge, tag publication, and GitHub Release publication are already complete for `v2.0.0`; this note documents the shipped artifact

## 2.x Roadmap
- Labs 模块真正产品化实现（Meet / VPN / Lumo / Wallet / Standard Notes parity 仅保留 preview 壳层 + route plumbing）
- `database-per-tenant` 升级（backend §11.1 第 2 项）
- Kafka / 独立 MQ 替换 Redis Streams（backend §11.1 第 5 项）
- Billing ledger 精细模型、外部支付编排（backend §11.1 第 3 项）
- Docs/Sheets 实时协作引擎重型升级（backend §11.1 第 4 项）
