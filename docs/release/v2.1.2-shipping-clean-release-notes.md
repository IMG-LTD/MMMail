# MMMail v2.1.2 Shipping Clean Release Notes

## Summary

`v2.1.2-shipping-clean` is the current repository baseline on `main`. It closes the v2.1.x migration cleanup by bringing the admin frontend into the repository, pruning development-only artifacts, and keeping the release gate as the shipping boundary.

## What Changed

- `frontend-admin` is the target product frontend for v2.1.x and later work.
- `frontend-v2` remains in the repository as a legacy contract and migration reference.
- Release validation is centralized through `scripts/release-gate.sh`.
- Docker-backed backend and frontend checks remain part of the release discipline.
- Repository cleanup removed development-only local artifacts from the tracked release surface.

## Current Boundaries

- MMMail Free self-hosting remains the public baseline.
- Real payment processing, merchant credentials, refunds, invoices, and license signing private keys are not included in the public repository.
- `frontend-v2` must not receive new product features.

## Validation

The release baseline was tagged at `v2.1.2-shipping-clean` after the shipping cleanup commit on `main`.
