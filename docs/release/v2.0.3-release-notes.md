# MMMail v2.0.3 Release Notes

Prepared as the mainline cleanup release after the shipped `v2.0.0` baseline and the `v2.0.1` / `v2.0.2` hardening slices.

## Highlights
- preserve the mixed legacy repository state on `archive/v2-only-pre-cleanup-20260423`
- simplify `main` to the `frontend-v2` runtime, validation, and documentation surface
- refresh install, upgrade, runbook, support-boundary, and release docs for the v2-only repository

## Validation
- `pnpm --dir frontend-v2 test`
- `pnpm --dir frontend-v2 typecheck`
- `mvn -f backend/pom.xml -pl mmmail-server -am -DskipTests compile`
- `bash scripts/validate-local.sh`

## Included
- `docker-compose.yml` and `docker-compose.minimal.yml` now build `frontend-v2/Dockerfile`
- `.env.example`, runtime validation, and CI use `VITE_API_BASE_URL` instead of legacy `NUXT_PUBLIC_*`
- release-facing docs now describe the current `frontend-v2 Web + Spring Boot + MySQL / Redis` mainline
- legacy `frontend/` and `community-v1` assets are preserved on the archive branch instead of remaining on `main`

## Upgrade Notes
- this release does not change the backend deployment topology or the published API base path
- the current production schema version remains `15`
- operators should re-read `.env.example`, `docs/ops/install.md`, and `docs/ops/upgrade.md` before their next deployment

## Known Limitations
- `v2.0.3` does not promote preview modules into `GA`
- `v2.0.3` does not add native clients, inbound mail, or commercial billing orchestration
