# MMMail v2.0.4 Release Notes

Prepared as the install onboarding and first-login guide release after the `v2.0.3` v2-only mainline cleanup.

## Highlights
- add one-click Bash and PowerShell installers for minimal and standard Docker Compose deployment paths
- restructure first-time install documentation around one-click, Docker manual, bare-metal manual, and local development paths
- add a browser-local first-login onboarding modal with four quick-start steps and a Settings reopen entry

## Validation
- `pnpm --dir frontend-v2 test`
- `pnpm --dir frontend-v2 typecheck`
- `bash scripts/validate-local.sh`
- `bash -n scripts/install.sh`
- MMMail CI on `main` for merge commit `dd3107023f60b9a1871c142b0bc00fcc15174514`

## Included
- `scripts/install.sh` and `scripts/install.ps1` wrap the existing minimal and standard Compose deployments with env bootstrapping and validation
- `README.md`, `docs/ops/install.md`, `docs/ops/install.en.md`, and `docs/deployment-runbook.md` now point first-time operators to the install path selector
- `frontend-v2` now includes browser-local onboarding state, localized quick-start content, a welcome modal, App-level first-login trigger, and a Settings Getting Started reopen panel
- contract tests cover install path documentation/scripts and the onboarding content/store/modal/App/settings integration

## Upgrade Notes
- this release does not change backend APIs, deployment topology, or the production schema version
- operators can continue existing deployments; first-time operators should start from `docs/ops/install.md`
- PowerShell installer syntax was not verified locally because `pwsh` was unavailable in the release environment

## Known Limitations
- browser/UI visual verification of the onboarding modal was not performed in the release environment
- this release does not add backend-synced onboarding progress or cross-device onboarding state
- this release does not promote preview modules into `GA`
