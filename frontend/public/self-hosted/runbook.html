MMMail Community v1.5 运维 Runbook / Operations Runbook

入口 / Entry Points

  • 健康检查 / Health: GET /actuator/health
  • Prometheus: GET /actuator/prometheus
  • 管理页 / Admin UI: /settings/system-health
  • API docs: /swagger-ui.html and /v3/api-docs
  • API quick page: /self-hosted/api.html
  • 本地门禁 / Local gate: bash scripts/validate-local.sh

日常检查 / Daily Checks

  • 确认前端可访问，且 /manifest.webmanifest 返回正常。
  • Confirm the frontend is reachable and /manifest.webmanifest resolves.
  • 管理员登录后检查设置页中的 PWA、Mail E2EE foundation、Adoption readiness 面板。
  • As an admin, review the PWA, Mail E2EE foundation, and Adoption readiness panels from Settings.
  • 如需接口排查，先打开 Swagger UI，再结合 OpenAPI JSON 进行请求比对。
  • For API debugging, inspect Swagger UI first and compare payloads against the raw OpenAPI JSON.

边界 / Boundary

  • 当前主线已包含 Mail E2EE 草稿 / 附件加密、密钥恢复、外部密码保护安全投递的附件闭环与草稿恢复、Drive owner/readable-share E2EE foundation、Web Push 与 SMTP outbound adapter。
  • The current mainline includes Mail E2EE draft and attachment encryption, key recovery, password-protected external secure delivery with encrypted attachments and draft reopen, Drive owner/readable-share E2EE foundation, Web Push, and the SMTP outbound adapter.
  • 零知识架构、完整 MIME 外部 E2EE、SMTP inbound / IMAP / Bridge 仍不属于当前已交付能力。
  • Zero-knowledge architecture, full MIME external E2EE interoperability, and SMTP inbound / IMAP / Bridge remain outside the currently shipped scope.
