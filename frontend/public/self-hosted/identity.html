MMMail Community v1.6.1

Identity Readiness / 身份就绪说明

This page is an honesty layer for self-hosted operators. Community v1.6.1 does not ship SSO, SCIM, or LDAP automation. What it does provide is a readiness path so those enterprise questions are answered clearly before rollout.

What readiness means / readiness 的含义

  • Document the current gap explicitly.
  • Prepare directory and fallback decisions early.
  • Avoid promising enterprise login or provisioning before it exists.

Operator checklist / 运维检查项

Identity ownership

List the current IdP, owned domains, and authoritative user source.

Fallback admins

Keep local break-glass administrators and document recovery ownership.

Provisioning expectation

Capture whether future lifecycle management must support create, suspend, rename, and transfer.

Current boundary / 当前边界

MMMail Community v1.6.1 still uses local MMMail accounts for sign-in and lifecycle. This page is planning material only. It is not an enterprise identity feature flag.
