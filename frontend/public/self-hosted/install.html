MMMail Community v1.6.1 自托管安装说明 / Self-hosted Install Guide

基线 / Baseline

  • 当前页面对应 Community v1.6.1 mainline continuation，已补齐 Suite 主线协作链路、默认导航中的 Pass Beta、收敛后的 Labs 默认目录，以及既有的 PWA、Mail E2EE、外部密码保护安全投递、Drive E2EE foundation、Web Push、SMTP outbound adapter 与 Calendar internal invitation orchestration。
  • This page follows Community v1.6.1 mainline continuation, including the Suite mainline journey, a visible Pass Beta entry in default navigation, the narrowed Labs catalog, plus the existing PWA, Mail E2EE loop, password-protected external secure delivery, Drive E2EE foundation, Web Push, SMTP outbound adapter, and Calendar internal invitation orchestration.
  • 仍不承诺 SMTP inbound / IMAP / Bridge、零知识邮件架构、完整 MIME 级外部 E2EE 或 Hosted 自动化 onboarding。
  • It still does not claim SMTP inbound / IMAP / Bridge, zero-knowledge mail, full MIME-grade external E2EE interoperability, or hosted onboarding automation.

安装步骤 / Install Steps

  • 复制 .env.example 到 .env，替换 JWT、数据库、Redis、Nacos 等占位值。
  • Copy .env.example to .env and replace placeholder values for JWT, database, Redis, and Nacos.
  • 执行 ./scripts/validate-runtime-env.sh .env，确认运行时配置完整。
  • Run ./scripts/validate-runtime-env.sh .env before booting the stack.
  • 使用 docker compose --env-file .env up -d --build 启动，并用 docker compose ps 检查服务状态。
  • Boot with docker compose --env-file .env up -d --build and confirm status via docker compose ps.

关键验证 / Verification

  • 前端：访问 http://127.0.0.1:3001，确认 /manifest.webmanifest 可访问，设置页能看到 PWA 与 Mail E2EE foundation 面板。
  • Frontend: open http://127.0.0.1:3001, confirm /manifest.webmanifest resolves, and check the Settings page for PWA, Mail E2EE, and Adoption readiness panels.
  • 采用入口：确认 /suite?section=overview 已显示 Mail → Calendar → Drive → Pass 主线，且 /pass 已进入默认导航。
  • Adoption entry: confirm /suite?section=overview shows the Mail → Calendar → Drive → Pass chain and /pass is visible in the default navigation.
  • 后端健康：curl -sf http://127.0.0.1:8080/actuator/health
  • API 文档：后端默认暴露 /swagger-ui.html 与 /v3/api-docs。
  • API docs: the backend exposes /swagger-ui.html and /v3/api-docs by default.
  • 浏览器内 adoption guide：/self-hosted/adoption.html
  • 浏览器内 API quick page：/self-hosted/api.html
