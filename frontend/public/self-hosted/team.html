MMMail Community v1.6.1

Team Enablement / 团队启用指引

This page turns the current Community mainline into a concrete team rollout sequence: Mail → Calendar → Drive → Pass. It is for self-hosted pilots and operators, not a hosted onboarding portal.

Who should use this / 适用角色

Platform admin

Owns security baseline, boundary messaging, and pilot scope.

Team lead

Turns the mainline into a repeatable handoff that the pilot team can actually use.

Pilot members

Validate the real collaboration path with actual files, events, and secrets.

Recommended rollout / 推荐 rollout

  1. Finish administrator baseline: Mail E2EE, PWA / Web Push, boundary review.
  2. Select a real pilot task instead of demo data.
  3. Use Mail to start the handoff, Calendar to schedule it, Drive to deliver the artifact, and Pass to close the loop.
  4. Require a second teammate to consume or confirm the handoff.
  5. Record the failure points before scaling beyond the pilot.

Proof checklist / 证据清单

  • One mainline mail thread
  • One calendar checkpoint or share record
  • One drive delivery or secure artifact
  • One pass secret handoff or secure link
  • One short pilot retro

Current boundary / 当前边界

Team enablement in Community v1.6.1 does not include enterprise identity automation, Bridge, IMAP, or non-mainline preview engines as rollout commitments.
