Runtime Architecture / 运行时架构

This page explains the runtime model that the current repository actually ships. It exists so self-hosted adopters and contributors do not confuse standard-mode dependencies with a delivered microservice platform.

Current runtime model / 当前运行模型

Frontend

One Nuxt 3 web application provides the workspace shell and product surfaces.

Backend

One Spring Boot backend process exposes the current Community API surface.

State

MySQL and Redis remain the default persistence and cache baseline.

Modes

Standard mode can include Nacos; minimal mode can disable it with MMMAIL_NACOS_ENABLED=false.

What standard mode means / 标准模式意味着什么

Standard mode currently adds a local Nacos dependency placeholder for configuration and discovery alignment. It does not mean the repository already ships a true microservice mesh, service gateway layer, or event-bus-driven runtime.

Layer Current status What it is not
Web shell Single Nuxt deployment Not multiple independently deployed frontend apps
Backend API Single Spring Boot runtime Not a split Spring Cloud service fleet
Nacos Optional standard-mode dependency Not proof of shipped service-discovery architecture
Kafka Only a reserved config placeholder in current runtime files Not an active, delivered message bus dependency in this repo

Minimal mode / 最小模式

  • Set MMMAIL_NACOS_ENABLED=false to remove Nacos from the local adoption path.
  • Use docker-compose.minimal.yml when the team wants the lightest self-hosted baseline for evaluation or local rollout.
  • Minimal mode lowers dependency weight; it does not claim a replacement config-center product or new distributed-runtime design.

Why this matters / 为什么要显式写清楚

  • Open-source adopters need to know whether they are evaluating a modular web suite or a distributed platform.
  • Contributors need a stable narrative before touching deployment docs, onboarding, or identity readiness work.
  • Enterprise evaluation should stay honest: current value is in the shipped mainline, not in implied infrastructure breadth.
